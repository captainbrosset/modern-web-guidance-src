Here's a summary of what was done to make navigations as instant as possible:

### Created files

- **`styles.css`** - Shared stylesheet extracted from the inline `<style>` block, now cached by the browser across all pages
- **`instant-nav.js`** - Client-side instant navigation engine
- **12 new HTML pages** - All pages linked from the original `index.html` (`menu`, `locations`, `rewards`, `account`, `order-online`, `about`, `careers`, `contact`, `privacy-policy`, `logout`, and menu sub-pages for `seasonal`, `espresso`, `cold-brew`)

### How instant navigation works

The `instant-nav.js` script implements three layers of prefetching:

1. **Viewport-based prefetching** - Uses `IntersectionObserver` to automatically fetch any link that scrolls into view (with a 200px margin), so by the time a user even thinks about clicking, the page is already cached.

2. **Hover/touch prefetching** - On `pointerenter` and `touchstart`, the target page is fetched immediately. Since there's typically 200-400ms between hover and click, the page is usually ready before the click lands.

3. **SPA-style DOM swap** - When a link is clicked, instead of a full page load, the script swaps the `<body>` content and updates the `<title>` and URL via `history.pushState`. This eliminates the browser's page teardown/rebuild cycle entirely.

Additional optimizations:
- The current page is cached on load so back-navigation is instant
- Browser back/forward is intercepted via `popstate` for instant history navigation
- The CSS is a shared external file, so it's cached once and reused across all pages
- The homepage uses `<link rel="prefetch">` hints for the 5 most likely navigation targets
- The script falls back to `.html` extension resolution for compatibility with static hosts
