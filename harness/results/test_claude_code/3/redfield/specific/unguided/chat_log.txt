Here's a summary of the changes made:

**What was removed:**
- The wrapping `<div class="container">` and its `position: relative` styling (no longer needed)
- All JavaScript — the `<script>` block with `mouseover`/`mouseout` event listeners is gone entirely
- Manual `display: none` / `display: block` toggling
- Absolute positioning relative to a parent container

**What was added:**
- **`popover` attribute** on the tooltip `<div>` — makes it a popover element managed by the browser (hidden by default, rendered in the top layer)
- **`interesttarget="tooltip"` attribute** on the `<button>` — this is the Interest Invokers API; it automatically shows the referenced popover on hover/focus and hides it on mouseleave/blur, with no JavaScript required
- **CSS Anchor Positioning** (`anchor-name: --trigger` on the button, `position-anchor: --trigger` + `position-area: top` on the tooltip) — positions the tooltip above the button declaratively, replacing the old `position: absolute; bottom: 125%` hack
- **`position-try-fallbacks: flip-block`** — if there's not enough space above, the browser will automatically flip the tooltip below the button

The result is a zero-JavaScript, fully declarative tooltip using modern web platform APIs. Note that Interest Invokers are a newer feature (Chrome 131+), so this requires a modern browser.
