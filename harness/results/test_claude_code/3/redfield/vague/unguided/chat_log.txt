Here's a summary of the modernization changes:

### What changed

**Removed: imperative JavaScript tooltip logic**
- The old implementation used `getElementById`, `mouseover`/`mouseout` listeners, and `style.display = 'block'/'none'` toggling.

**Replaced with: pure CSS tooltips**
- Tooltips now show/hide via `:hover` and `:focus-within` pseudo-classes on a `.tooltip-wrapper` parent -- no JS required for core functionality.
- Uses `opacity` + `visibility` + `translate` transitions for a smooth fade-and-slide animation instead of the abrupt `display` toggle.

**Modern CSS features throughout:**
- **Custom properties** (`--color-*`, `--radius-*`, `--shadow-*`, `--transition-*`) for a consistent, themeable design system.
- **Logical properties** (`inline-size`, `block-size`, `inset-block-start`, `margin-inline-start`, etc.) for better internationalization support.
- **Modern selectors** (`:has()`, `:focus-visible`, `:focus-within`, `[role="tooltip"]` attribute selectors).
- **`translate` property** (the individual transform, not `transform: translate()`) for cleaner animation.

**Accessibility:**
- Tooltip elements use `role="tooltip"` with matching `aria-describedby` on trigger buttons so screen readers announce tooltip content.
- Keyboard users can trigger tooltips via `:focus-within` / `:focus-visible`.
- Pressing **Escape** dismisses focused tooltips (the only remaining JS, as progressive enhancement).

**Expanded demo page:**
- **Tooltip positions** section -- top, bottom, left, right.
- **Inline tooltips** section -- tooltips on links and `<abbr>` elements within running text.
- **Card grid** section -- photo cards (Unsplash stock images) with icon-button actions (Like, Share, Save), each with tooltips. Demonstrates real-world usage with `loading="lazy"` images and a responsive CSS grid.
