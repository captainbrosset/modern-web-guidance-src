Here's a summary of the modernization:

### What changed

**Before** — The old implementation used imperative JavaScript (`mouseover`/`mouseout` event listeners toggling `display: block/none`) with a wrapper `<div class="container">` for relative positioning.

**After** — The new implementation uses three modern web platform features:

1. **Popover API** (`popover="hint"`) — The tooltip is now a native popover element. The browser handles show/hide lifecycle, top-layer rendering, and light-dismiss behavior. No JavaScript for visibility toggling.

2. **Interest Invokers** (`interestfor="tooltip"`) — The button declaratively declares its interest target. The browser shows the popover on hover/focus automatically — no event listeners needed.

3. **CSS Anchor Positioning** (`anchor-name` / `position-anchor` / `position-area`) — The tooltip is tethered to the button using CSS anchor positioning instead of manual `position: absolute` + `bottom: 125%` + `transform` hacks. `position-try-fallbacks: flip-block` automatically flips the tooltip below the button if there's no room above.

**Polyfills** — All three features have limited browser support today, so the implementation conditionally loads local polyfills only when the browser lacks native support:
- `polyfills/interestfor.min.js` — Interest Invokers
- `polyfills/popover.min.js` — Popover API
- `polyfills/css-anchor-positioning.js` — CSS Anchor Positioning

The wrapper `<div class="container">` was removed since anchor positioning eliminates the need for a positioned parent.
