The landing page is complete. Here's a summary of what was built in `index.html`:

### Page Structure
- **Fixed navbar** with blur backdrop, linking to Menu / About / Visit sections
- **Full-viewport hero** with headline, subtitle, and two CTA buttons
- **Menu section** with a 3-card grid (Espresso, Oat Milk Latte, Cold Brew) using Unsplash stock photos
- **About section** (dark background) with a 3-column value proposition grid
- **Visit/CTA section** with address and hours
- **Footer**

### Three Key Features

1. **Non-modal Popover on Hover** — The "Details" button in the hero uses the native **Popover API** (`popover` attribute + `showPopover()`/`hidePopover()`). It's triggered via `pointerenter`/`pointerleave` events with logic to keep the popover open while the pointer moves between the button and the popover. CSS `@starting-style` and `allow-discrete` transitions animate the popover in and out. It's anchored to the button using **CSS Anchor Positioning** (`anchor-name` / `position-anchor`).

2. **Scroll-driven hero animation** — The hero image uses **CSS Scroll-Driven Animations** (`animation-timeline: view()`) with `animation-range: entry 0% entry 60%`. As the image enters the viewport, it fades from `opacity: 0` to `0.7` and scales down from `1.15` to `1`. The menu cards also use the same technique for a fade-up effect.

3. **Adaptive Loading (Network Information API)** — On page load, the script checks `navigator.connection.effectiveType` and `saveData`. On slow networks (`2g`, `slow-2g`, or Save-Data enabled), the hero `<img>` src is swapped to an 80px-wide, q=20 placeholder with a CSS blur filter applied. A "Load full image" button is injected so the user can opt-in to download the full-resolution image.
