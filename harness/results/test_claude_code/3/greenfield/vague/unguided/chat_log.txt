The landing page is built. Here's what's in `index.html`:

### Page structure
- **Fixed navbar** with logo ("Ember & Brew") and anchor links to Menu / About / Visit
- **Hero section** with headline, tagline, CTA button, and a large feature image
- **Menu section** with 3 drink cards (Espresso, Vanilla Oat Latte, Signature Mocha)
- **About section** with shop interior photo and story text
- **Footer** with branding

### Feature 1: Details button with ingredients tooltip
Each menu card has a **"Details" button**. Hovering over it reveals a tooltip that slides up with:
- Ingredient list specific to each drink
- Allergen information
- Smooth CSS transition (opacity + translateY)
- Arrow pointer via `::after` pseudo-element

### Feature 2: Scroll-driven hero image animation
A `scroll` event listener continuously calculates how far the hero image wrapper has entered the viewport:
- **Opacity** fades from 0 to 1
- **Scale** grows from 0.85 to 1.05
- Uses `{ passive: true }` for scroll performance

### Feature 3: Low-quality image placeholder (LQIP) for slow networks
Two layers work together:
1. **Inline SVG placeholder** — a blurred, coffee-colored shape rendered instantly (no network request) via a `data:` URI background. It's hidden with a fade-out once the real image fires its `load` event.
2. **Network Information API** (`navigator.connection.effectiveType`) — on `slow-2g`/`2g` connections the hero image URL is swapped to `w=400&q=40`, and on `3g` to `w=720&q=60`, reducing payload significantly. On `4g` or when the API isn't available, the full `w=1080&q=80` image loads.

All images are sourced from Unsplash with `loading="lazy"` attributes for native lazy loading.
