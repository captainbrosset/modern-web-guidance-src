The landing page is built. Here's a summary of what's in `index.html`:

### Structure
- **Nav** — fixed top bar with logo and anchor links (Menu, About, Visit)
- **Hero** — headline, tagline, CTA button, and a large hero image
- **Signature Drinks** — 3-card grid (Velvet Latte, Cold Brew Tonic, Honey Cortado) with Unsplash photos
- **About / Visit / Footer** — story section, address, and copyright

### Key Features

**1. Popover API with hover trigger**
- Each drink card has a `<button class="details-btn">` with `popovertarget` pointing to a `<div popover>` containing the ingredients list.
- The popover is **non-modal** (`popover="auto"` is the default) — it doesn't block interaction with the rest of the page.
- JavaScript adds `mouseenter`/`mouseleave` listeners on both the button and the popover element, calling `showPopover()` / `hidePopover()` so it opens on hover (with a 200ms grace period to avoid flicker when moving between the button and popover).
- CSS `@starting-style` provides a smooth fade+slide entrance animation.

**2. Scroll-driven animation on the hero image**
- Uses the native CSS `animation-timeline: view()` spec behind `@supports (animation-timeline: view())`. The `hero-reveal` keyframe animates from `opacity: 0; scale(0.85)` to `opacity: 1; scale(1)` as the element enters the viewport (`animation-range: entry 0% entry 60%`).
- For browsers that don't support scroll-driven animations, an `IntersectionObserver` fallback toggles an `.in-view` class that triggers an equivalent CSS transition.

**3. Adaptive loading via the Network Information API**
- Checks `navigator.connection` for `saveData`, `effectiveType` (slow-2g/2g/3g), and `downlink < 1.5 Mbps`.
- On slow networks: loads a tiny 80px-wide, q=20 placeholder version of the hero image and shows a "Lite mode active" banner.
- On fast networks: loads the full 1200px-wide, q=80 image.
- An SVG coffee-cup placeholder is shown over the image area until the chosen image finishes loading.
- Listens for the `change` event on `navigator.connection` to upgrade the image dynamically if the connection improves.
