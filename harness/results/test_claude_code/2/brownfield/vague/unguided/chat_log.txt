Everything is in place. Here's a summary of what was done:

## Changes Made

### Problem
The site was a single `index.html` with navigation links pointing to pages that didn't exist, meaning every click would result in a 404.

### Solution — Three layers of instant navigation

**1. Created all 13 linked pages** as `directory/index.html` files for clean URLs:
- `/menu`, `/menu/seasonal`, `/menu/espresso`, `/menu/cold-brew`
- `/locations`, `/rewards`, `/account`
- `/about`, `/careers`, `/contact`, `/privacy-policy`
- `/order-online`, `/logout`

**2. Extracted shared CSS** into `/styles.css` — loaded once and cached by the browser across all pages, so subsequent navigations don't re-download styles.

**3. Created `/instant-nav.js`** with a three-layer prefetching strategy:

| Layer | Mechanism | Browser Support | Effect |
|-------|-----------|----------------|--------|
| **Speculation Rules API** | `speculationrules` script | Chrome 109+ | Full **prerender** of pages on hover — the target page is loaded and rendered in a hidden tab before the user clicks, making navigation effectively instant (0ms perceived load) |
| **Prefetch on hover/focus** | `<link rel="prefetch">` injected on `pointerenter`/`focusin` | All modern browsers | Falls back to fetching the HTML document into the browser cache when the user hovers a link |
| **Idle prefetch of visible links** | `IntersectionObserver` + `requestIdleCallback` | All modern browsers | When the browser is idle, prefetches all same-origin links visible in the viewport |

The `/logout` path is excluded from prerendering via the speculation rules to avoid triggering a logout action prematurely.
