Here's what I added to `index.html:200-215`:

```html
<script type="speculationrules">
  {
    "prerender": [
      {
        "source": "document",
        "where": {
          "and": [
            { "href_matches": "/*" },
            { "not": { "href_matches": "/logout" } }
          ]
        },
        "eagerness": "moderate"
      }
    ]
  }
</script>
```

**How it works:**

- **Speculation Rules API** — This is the native browser API for speculative prerendering. The `<script type="speculationrules">` block is declarative; no JavaScript execution is needed.
- **`"source": "document"`** — Tells the browser to find matching links within the page itself rather than listing URLs explicitly. This automatically covers `/menu`, `/locations`, `/rewards`, `/account`, and any other same-origin links in the page.
- **`"not": { "href_matches": "/logout" }`** — Explicitly excludes `/logout` from prerendering, preventing accidental session termination.
- **`"eagerness": "moderate"`** — Prerenders on hover/pointer-down intent rather than immediately, which balances speed improvement against resource usage.
- **Graceful degradation** — Browsers that don't support the Speculation Rules API simply ignore the `<script type="speculationrules">` tag and navigate normally, so there's zero downside.
