Here's a summary of the changes made:

**Removed:**
- The wrapping `.container` div (no longer needed — positioning is handled by CSS Anchor Positioning)
- All JavaScript (`<script>` block with `mouseover`/`mouseout` event listeners and manual `style.display` toggling)
- Manual `display: none` / `position: absolute` / `bottom: 125%` / `left: 50%` / `transform: translateX(-50%)` positioning hacks

**Added:**
- `interestfor="tooltip"` on the `<button>` — this is the **Interest Invokers** attribute that declaratively tells the browser to show the referenced popover when the user "shows interest" (hover or focus)
- `popover="hint"` on the tooltip `<div>` — the `hint` popover type is specifically designed for tooltips/hovercards, with built-in light-dismiss and appropriate stacking behavior
- **CSS Anchor Positioning** (`anchor-name`, `position-anchor`, `position-area`, `position-try-fallbacks`) to position the tooltip above the button, with automatic fallback to flip below if there's not enough space

**Key benefits:**
- **Zero JavaScript** — the browser handles show/hide timing, hover delays, and state management
- **Keyboard accessible** — focus on the button shows the tooltip automatically
- **Touchscreen accessible** — the UA provides appropriate interaction for touch users
- **Light dismiss** — clicking elsewhere or pressing Escape closes the tooltip
- **Proper stacking** — `popover=hint` uses the top layer, so no `z-index` battles

Note: This uses two cutting-edge platform features. `interestfor` with `popover=hint` is currently behind flags in Chromium (Chrome Canary). CSS Anchor Positioning is shipped in Chrome 125+.
