Here's a summary of all the changes made to `index.html`:

**HTML changes:**
- Removed the wrapper `<div class="container">` — no longer needed since positioning is handled by CSS Anchor Positioning instead of `position: relative` on a parent
- Added `interestfor="tooltip"` to the button — this declaratively links the trigger to the popover, replacing the manual `mouseover`/`mouseout` event listeners
- Added `popover="hint"` to the tooltip `<div>` — this uses the Popover API's `hint` type, which is designed specifically for tooltips (auto-dismissing, non-modal)

**CSS changes:**
- Replaced manual absolute positioning (`position: absolute; bottom: 125%; left: 50%; transform: translateX(-50%)`) with **CSS Anchor Positioning**:
  - `anchor-name: --tooltip-trigger` on the button
  - `position-anchor: --tooltip-trigger` on the tooltip
  - `position-area: top` to place it above the trigger
  - `width: max-content` and `margin-inline: auto` for natural sizing
  - `position-try-fallbacks: flip-block` so it flips below the button if there's not enough space above
- Added `border: none` to remove the default popover border
- Removed `display: none` and `z-index` — the Popover API handles visibility and stacking context automatically

**JavaScript changes:**
- Removed all imperative event listener code (`addEventListener` for `mouseover`/`mouseout`)
- Added conditional polyfill loading for three features:
  - **Interest Invokers** — detected via `HTMLButtonElement.prototype.hasOwnProperty("interestForElement")`
  - **Popover API** — detected via `HTMLElement.prototype.hasOwnProperty("popover")`
  - **CSS Anchor Positioning** — detected via `"positionAnchor" in document.documentElement.style`
