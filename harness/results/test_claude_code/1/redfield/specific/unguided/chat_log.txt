Here's what changed:

**Removed:**
- The wrapping `.container` div (no longer needed for positioning)
- All JavaScript — no `mouseover`/`mouseout` event listeners, no manual `style.display` toggling
- Manual `display: none` / `position: absolute` positioning logic

**Added:**
- `popover="hint"` on the tooltip `<div>` — the Popover API manages visibility automatically, and `hint` is the correct popover type for tooltip/hovercard semantics (auto-dismisses, only one hint visible at a time)
- `interestfor="tooltip"` on the `<button>` — this is the Interest Invokers attribute that declaratively wires up hover/focus to show the popover, and de-hover/blur to hide it. No JS needed.
- CSS Anchor Positioning (`anchor-name: --trigger` on the button, `position-anchor` and `position-area` on the tooltip) to position the tooltip above the button, replacing the old manual absolute positioning
- `interest-delay-start: 0.3s` and `interest-delay-end: 0.2s` to control the hover-in and hover-out delays

The browser now handles all of the complexity: mouse hover/de-hover with configurable delays, keyboard focus/blur activation, touchscreen long-press context menu integration, and accessibility (the hint popover's text is automatically exposed as the button's accessible description).
