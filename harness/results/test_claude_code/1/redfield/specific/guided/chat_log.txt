Here's a summary of the changes made:

**What was removed:**
- The wrapping `<div class="container">` with `position: relative` (no longer needed)
- All JavaScript event listeners (`mouseover`/`mouseout`) and manual DOM manipulation (`tooltip.style.display`)
- Manual absolute positioning with `bottom: 125%`, `left: 50%`, `transform: translateX(-50%)`
- `display: none` / `display: block` toggling
- `z-index` management

**What was added:**

1. **HTML Popover API** - The tooltip uses `popover="hint"` which gives it built-in show/hide behavior, top-layer rendering, and light-dismiss semantics without any JavaScript.

2. **Interest Invokers** - The button uses the `interestfor="tooltip"` attribute to declaratively link hover/focus interest to the popover, replacing the manual `mouseover`/`mouseout` listeners entirely.

3. **CSS Anchor Positioning** - The button declares `anchor-name: --tooltip-trigger` and the tooltip uses `position-anchor` and `position-area: top` to tether itself above the button. `position-try-fallbacks: flip-block` automatically flips the tooltip below if there's not enough space above.

4. **Polyfills with feature detection** - Since these APIs have limited browser availability, the script conditionally loads local copies of polyfills for Interest Invokers, Popover, and Anchor Positioning only when the browser lacks native support.
