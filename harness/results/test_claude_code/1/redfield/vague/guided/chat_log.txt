Here's a summary of the modernization changes:

**Before (imperative approach):**
- Tooltip visibility manually toggled via JS `mouseover`/`mouseout` event listeners setting `display: block/none`
- Positioned with `position: absolute` + manual `bottom`/`left`/`transform` offsets inside a wrapper `<div class="container">`
- No fallback if positioned off-screen

**After (declarative approach):**
1. **Popover API** (`popover="hint"`) — the tooltip is now a native popover, so the browser manages its visibility and top-layer stacking. No JS visibility logic needed.

2. **Interest Invokers** (`interestfor="tooltip"`) — declaratively links the button to the popover on hover/focus interest. Replaces the manual `mouseover`/`mouseout` listeners entirely.

3. **CSS Anchor Positioning** (`anchor-name` / `position-anchor` / `position-area`) — the tooltip is tethered to the button using the new anchor positioning spec. `position-try-fallbacks: flip-block` automatically flips the tooltip below the button if there isn't enough space above.

4. **Conditional polyfills** — since all three features have limited or newly-available baseline status, polyfills are loaded only when the browser lacks native support:
   - `interestfor` polyfill for Interest Invokers
   - `@oddbird/popover-polyfill` for Popover API
   - `@oddbird/css-anchor-positioning` for CSS Anchor Positioning

The wrapper `<div class="container">` was removed since anchor positioning eliminates the need for a `position: relative` container. The entire implementation now has **zero imperative JavaScript** for tooltip behavior.
