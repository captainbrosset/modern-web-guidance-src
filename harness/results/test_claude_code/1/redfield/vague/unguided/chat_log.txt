Here's a summary of the modernization changes:

**Removed**
- All imperative JavaScript (the `<script>` block is gone entirely)
- Hardcoded `#tooltip` / `#tooltip-trigger` IDs
- `display: none/block` toggling from JS

**Added — CSS**
- **Pure-CSS show/hide** via `:hover` and `:focus-within` on a `.tooltip-wrap` container — no JS needed
- **CSS custom properties** (`--tt-bg`, `--tt-fg`, `--tt-radius`, etc.) for easy theming
- **Smooth transitions** — opacity fade + subtle slide, with a `prefers-reduced-motion` media query that disables animation for users who prefer it
- **Four placement options** (`top`, `bottom`, `left`, `right`) controlled by a `data-position` attribute
- **`max-content` width** with a `max-width` cap so tooltips size naturally

**Added — Accessibility**
- `role="tooltip"` on every tooltip element
- `aria-describedby` linking each trigger to its tooltip
- **Keyboard support** via `tabindex="0"` on non-button triggers and `:focus-within` to show tooltips when tabbing
- Semantic `<section>` grouping

**Added — Rich content demo**
- An **image card tooltip** using an Unsplash stock photo, showing that tooltips can contain structured HTML (image + title + description)
- An **inline info icon** (`?` circle) demonstrating tooltips on non-button elements

**Architecture**
- The `.tooltip-wrap` pattern is fully reusable — just wrap any trigger + `[role="tooltip"]` element and optionally set `data-position`. No IDs or JS wiring required.
