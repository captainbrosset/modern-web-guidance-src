I added a `<script type="speculationrules">` block at the end of `<body>` in `index.html:210-222`. This uses the **Speculation Rules API** — a native web platform feature for pre-rendering pages before the user navigates to them.

**How it works:**

- **`"source": "document"`** — automatically discovers links in the page's DOM rather than requiring a hardcoded URL list.
- **`"href_matches": "/*"`** — matches all same-origin links (covering `/menu`, `/locations`, `/rewards`, `/account`, and any others).
- **`"not": { "href_matches": "/logout" }`** — explicitly excludes `/logout` from pre-rendering, preventing accidental session termination.
- **`"eagerness": "moderate"`** — pre-renders on hover/pointer-down intent signals, balancing speed gains against resource usage (as opposed to `"immediate"` which would speculatively pre-render everything on page load).

**Browser support:** Browsers that don't support the Speculation Rules API simply ignore the `<script type="speculationrules">` tag, so there's no fallback needed — navigation works normally in all browsers.
