Efficient Background Processing

The Challenge: Heavy DOM components, Javascript complex animations, or expensive data-polling often run continuously whether the user is looking at them or not. This burns battery life, ties up the main thread, and degrades overall page performance.

The Solution: By applying content-visibility: auto to a container, the browser fires a contentvisibilityautostatechange event indicating if the element's rendering is currently "skipped". We can listen to this event to safely pause our intense Javascript loops or HTTP polling when the user can't see the component anyway!
